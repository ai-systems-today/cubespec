Sixteen worked notebooks that mirror **every control on every tab** of the
[Sensitive-Spark dashboard](https://app.ai-systems.today), so you can
read the thesis chapter, open the dashboard in one tab and a notebook in
another, and watch the same numbers come out of both.

Each notebook follows the same structure:

1. **Abstract** — what the notebook proves and which dashboard surface it mirrors.
2. **Bootstrap** — `try: import cubespec / except: pip install …` (Colab-safe).
3. **Method** — the maths in 2–3 sentences.
4. **Code → plot → interpretation** triplets.
5. **Try this** — a small exercise inviting the reader to change a knob.
6. **Reproducibility footer** — version stamps and a hash of the result.

## For Madina (or any lab partner) handing back measurements

Start at **[notebook 10](10_doe_measurements.ipynb)**.

It is self-contained:

1. Generate a DOE design (notebook 06 or 07, or the dashboard's DOE tab).
2. Fill in `examples/madina_template.csv` with your replicate measurements.
3. Run notebook 10 — point its `MEAS_PATH` cell at your file.
4. Read the comparison summary, the predicted-vs-measured scatter, the
   residual diagnostics (RMSE, MAE, bias, R², Q-Q) and the
   measured-effects-vs-predicted-effects tornado.

The CSV format is identical to the one the dashboard's DOE tab accepts.

## Reproducibility

All notebooks pin `seed = 1337` (or another seed declared in cell #2). For
parity with the dashboard run notebook 16, which asserts bit-for-bit
equality of the surrogate output array against a fixture written by the
TypeScript dashboard.

## Authoring conventions

- Every notebook starts with a title, an abstract, a badge row and a
  bootstrap cell — all generated by `python scripts/normalise_notebooks.py`.
- Every notebook ends with a reproducibility footer (also normalised).
- New notebooks should start from [`_template.ipynb`](_template.ipynb).
- Keep cells small and prefer one plot per cell.

## In CI

The `.github/workflows/notebooks.yml` workflow executes every notebook in
this directory end-to-end on every push using `nbmake`. A failed cell
breaks the build.
