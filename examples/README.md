# Example data sets for `cubespec`

These CSVs are referenced by the notebooks (`python/notebooks/`) and the
plot scripts (`python/examples/plots/`).

| File | Purpose | Used by |
|---|---|---|
| `default_csp.csv` | Reference CSP (means + sds) matching the dashboard defaults. | All notebooks, all CLI examples. |
| `high_load_scenario.csv` | Same CSP but with `P4_Fx` mean shifted to 1.5 MN — illustrates how shifting a single input mean propagates through P9. | `02_doe_full_vs_fractional.ipynb`, `plot_p9_histogram.py`. |
| `correlation_geometry.csv` | 7×7 correlation matrix that only correlates the three geometry factors (`P1_dx`, `P2_dy`, `P3_dz`). | `01_quickstart.ipynb`, `04_sobol_sensitivity.ipynb`. |
| `correlated_geometry_full.csv` | Extended correlation matrix that adds a small density–geometry coupling and a tiny load-axis cross-talk. Documented inline. | `03_rsm_contour.ipynb`. |
| `cube_lab_measurements.csv` | **Synthetic** measured-vs-predicted dataset, 30 runs, generated from the surrogate with σ ≈ 1 MPa noise. Clearly labelled as synthetic in the file header. | `plot_measured_vs_predicted.py`. |

## Loading

```python
from cubespec import load_csp_csv
csp = load_csp_csv("python/examples/default_csp.csv")
```

Correlation CSVs use the convention: header row + index column with the
seven `P*` keys, body is a symmetric matrix with `1.0` on the diagonal.
Load with:

```python
import pandas as pd
corr = pd.read_csv("python/examples/correlation_geometry.csv", index_col=0).to_numpy()
```
